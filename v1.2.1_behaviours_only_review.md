## Resolved source-only decisions

| Source-only file | Resolution | Reason |
| --- | --- | --- |
| `entities/vanilla/enderman.json` | Ported to `BOB_behaviors/entities/vanilla/enderman.json` with the source-only fixes. | The global Enderman override is now approved for current main. Teleporting remains scoped to the not-riding component group, dragon-spawned Endermen can use `better_on_bedrock:dragon_summoned` to suppress loot/XP, the Ender Dragon summon applies that event, and bounty progress ignores the marked dragon-spawned variant. |
| `entities/vanilla/shulker.json` | Cherry-picked the shulker-armour targeting grouping into `BOB_behaviors/entities/end_mobs/shulker.json`. | Current main keeps its BOB-specific target priority, radius, persistence, reevaluation, and separate container-open targeting behaviour while adding the source-only armour targeting behaviour. |
| `items/ingots/stardust/stardust_block.json` | Do **not** port. | Current main's `blocks/other/ores/stardust_block.json` already declares `better_on_bedrock:stardust_block` with a construction creative-menu category, so it has the expected block-backed item form. The stardust-block recipe outputs that same identifier, and the ingot-from-block recipe consumes it back into 9 `better_on_bedrock:stardust_ingot`. Adding the v1.2.1 same-identifier `minecraft:item` block placer would be redundant and risks an identifier conflict. |

| `trading/goblin-trade-table.json` | Retained current branch. | Confirmed the previous review note still says this does not need to be changed.
