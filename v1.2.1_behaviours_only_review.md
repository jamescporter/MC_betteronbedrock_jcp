# v1.2.1 behaviour-pack audit and source-only review

This review now covers the full `v1.2.1_src_only/behaviors` tree, not just the files absent from `BOB_behaviors`. The refreshed audit compares every v1.2.1 behaviour-pack file against the current main behaviour pack by exact relative path and byte content.

## Audit method

- Enumerated every file in `v1.2.1_src_only/behaviors` and compared it with `BOB_behaviors/<same-relative-path>`.
- Counted exact matches, same-path changed files, and files that exist only in the v1.2.1 source tree.
- Also checked the reverse direction so current main-only behaviour-pack files are not mistaken for missing v1.2.1 work.
- For the remaining source-only files, searched the current main behaviour pack (`BOB_behaviors`) for candidate identifiers, item/block/entity identifiers, and related route or usage points.
- For same-path changed files, grouped the tasks by pack area so the next pass can review changes intentionally instead of copying v1.2.1 over current main.

## Detailed source-only candidate notes

### 1. `recipes/guidebook.json`

- **Source content:** shapeless crafting-table recipe `better_on_bedrock:guidebook` using `minecraft:dirt` and producing `better_on_bedrock:guide_book`.
- **Current references:** `BOB_behaviors/scripts/commands/guidebook.js` uses the same `better_on_bedrock:guidebook` name for the custom command and grants `better_on_bedrock:guide_book`; the item and placeable guide-book entity already exist.
- **Risk:** the recipe identifier collides semantically with the command identifier, but recipe identifiers and command names live in different systems. The real risk is gameplay economy: one dirt for the guide book looks intentionally cheap for onboarding or possibly a debug placeholder.
- **Recommendation:** import only if the desired design is that players can craft the guide book directly. If imported, consider reviewing the ingredient before merging.

### 2. `entities/vanilla/shulker.json`

- **Source content:** a vanilla `minecraft:shulker` override at `format_version` `1.21.70`, including shulker-armour filters that stop shulkers targeting fully equipped players.
- **Current references:** current BP already overrides `minecraft:shulker` at `BOB_behaviors/entities/end_mobs/shulker.json`, and armour recipes/items reference the shulker-armour identifiers.
- **Current-path conflict:** this is not absent functionally; it is absent only by exact relative path. The current shulker file has a different path, older format version, variant component groups, and additional custom targeting behaviour such as reacting to open containers.
- **Recommendation:** do not copy the source-only file into `entities/vanilla/`. If the v1.2.1 changes are desired, cherry-pick specific differences into the current `entities/end_mobs/shulker.json` after testing.

### 3. `entities/vanilla/enderman.json`

- **Source content:** global vanilla `minecraft:enderman` override with health, attack, teleport, looked-at, anger/calm, riding, and shulker-armour filters.
- **Current references:** current BP references `minecraft:enderman` in spawn rules, Ender Dragon behaviour, bounty scripting, and loot tables, but does not currently include an Enderman entity definition.
- **Risk:** importing this file would replace vanilla Enderman behaviour pack-wide. It is potentially valuable if the shulker armour set is meant to protect against Endermen, but it also changes base mob tuning and AI ownership.
- **Recommendation:** treat as a design review item, not a missing-file restore. If imported, validate Enderman spawning, anger state transitions, player stare detection, water/lava hurt behaviour, and bounty counting.

### 4. `items/ingots/stardust/stardust_block.json`

- **Source content:** item definition `better_on_bedrock:stardust_block` with display name, fire resistance, and `minecraft:block_placer` for block `better_on_bedrock:stardust_block`.
- **Current references:** current BP already defines block `better_on_bedrock:stardust_block`; shaped and shapeless recipes also output/consume that identifier.
- **Risk:** because the block identifier already exists, adding a same-identifier item may be redundant or may alter block-item behaviour. This needs in-game validation rather than blind restoration.
- **Recommendation:** investigate only if the stardust block is not appearing/placing correctly as an inventory item. Otherwise leave absent.

## Overall recommendation

Do not blindly restore anything from the v1.2.1 behaviour-pack tree. Most files already match current main exactly, but **129** same-path files still differ and need targeted review by subsystem. Only **4** files remain source-only by exact path: the guidebook recipe, Shulker override, Enderman override, and Stardust Block item. The guidebook recipe is the only simple gameplay import candidate; the Enderman override is a broad design decision; the Shulker override should be cherry-picked rather than copied; and the Stardust Block item should be investigated only if the existing block-item behaviour is faulty.
