## Ambience, fog, jukebox, and startup validation status

The ambience/fog/jukebox/startup script review is code-complete against v1.2.1 and current-main expectations for this batch. Current main keeps the player-sliced high/low cadence loops, proximity cache, quest batching, and ambient entity/fog movement thresholds instead of reverting to every-player/every-tick work. The v1.2.1 startup ordering is retained by deferring `BlockTypes.getAll()` until the script tick and registering custom block/item components and commands during `system.beforeEvents.startup`; `scripts/functionality/enchanted/main.js` is no longer imported directly from `scripts/main.js` because the entity-hit event already imports it and the module has no startup side effects.

Ambience remains intentionally current-main optimised: tracked ambient entities are reused, duplicate/stale entities are cleaned up, player leave/spawn/dimension-change state is reset, and the slow main loop now allows one cleanup call after `pog:ambientSounds` is removed. End fog remains intentionally current-main optimised: the 30-tick loop only processes tagged End players plus tracked players needing cleanup, caches player position/dimension/biome state, removes fog on dimension/tag loss, and avoids replaying the same fog/music transition. Jukebox handling keeps the v1.2.1 custom-record stop list while removing unused imports/locals and sharing the asynchronous command loop. The legacy property-based enchantment bridge now keeps the v1.2.1 invalid-entity guard and vector knockback fix with a compatibility fallback for older `applyKnockback` signatures.

In-game validation is still requested for new world load, existing world load, player join, dimension travel, jukebox use/break, enchantment hits, End biome fog transitions, leaving the End/removing `endFog`, toggling `pog:ambientSounds`, and long-session tick stability.
